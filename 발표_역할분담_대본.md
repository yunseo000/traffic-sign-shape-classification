## PPT 구성안

### 1장. 제목

- 교통 표지판 형태 분류 프로젝트
- Computer Vision 기반 이미지 분류 실험
- 조원: 오영덕, 김진식, 김수민, 최윤서

### 2장. 주제 선정 이유와 문제 정의

- 교통 표지판 인식은 자율주행과 운전자 보조 시스템에서 중요한 기술
- 실제 표지판 인식은 색상, 글자, 아이콘, 촬영 각도 등 요소가 많아 복잡함
- 본 프로젝트에서는 난이도를 조절하여 표지판의 기본 형태 분류에 집중
- 목표: 이미지를 보고 원형, 삼각형, 사각형 중 하나로 분류

### 3장. 데이터 소개

- 데이터 종류: 직접 생성한 합성 이미지 데이터

- 총 이미지 수: 480장
- 각 클래스별 이미지 수: 160장
- 이미지 크기: 64x64 PNG

### 4장. 데이터 생성 및 전처리

- 위치, 크기, 회전, 밝기, 배경 노이즈를 무작위로 적용
- 회색조 변환
- 픽셀 값 정규화
- 64x64 이미지를 32x32 특징으로 축소
- 학습 데이터 360장, 테스트 데이터 120장으로 분리

### 5장. 모델 구조

- 은닉층 1개를 가진 간단한 신경망 사용
- 입력: 32x32 이미지 특징
- 출력: circle, triangle, square 클래스 확률
- Softmax로 가장 확률이 높은 클래스를 최종 예측

### 6장. 코드 흐름

- `generate_dataset.py`: 이미지 데이터 생성
- `features.py`: 이미지 전처리 및 특징 추출
- `train.py`: 모델 학습과 평가
- `predict.py`: 새 이미지 예측
- `outputs/`: 결과 파일 저장

### 8장. 결과 해석

- 삼각형은 꼭짓점과 방향성이 뚜렷해 안정적으로 분류됨

먼저 주제 선정 이유를 설명드리겠습니다. 교통 표지판 인식은 자율주행 자동차나 운전자 보조 시스템에서 중요한 기술입니다. 자동차가 도로 위를 주행할 때 카메라를 통해 주변 환경을 인식하고, 표지판 정보를 이해해야 안전한 주행 판단을 할 수 있기 때문입니다.

하지만 실제 교통 표지판 인식은 생각보다 복잡합니다. 표지판의 색상, 내부 아이콘, 글자, 촬영 각도, 조명, 배경 등 다양한 요소를 고려해야 합니다. 예를 들어 같은 표지판이라도 밤에 찍힌 사진과 낮에 찍힌 사진은 밝기와 배경이 다르고, 멀리서 찍히거나 기울어진 경우에는 형태도 다르게 보일 수 있습니다.

그래서 저희는 프로젝트 난이도를 조절하기 위해 실제 표지판 전체 인식이 아니라, 가장 기본적인 단계인 형태 분류에 집중했습니다. 즉, 이미지를 보고 이 표지판이 원형인지, 삼각형인지, 사각형인지 분류하는 문제로 정의했습니다.

이 주제를 통해 컴퓨터 비전 프로젝트의 기본 흐름인 데이터 준비, 전처리, 모델 학습, 평가를 모두 경험할 수 있다고 판단했습니다. 또한 결과가 혼동행렬이나 예측 이미지로 시각화하기 쉬워서 발표와 보고서로 설명하기에도 적합하다고 생각했습니다.

### 김진식 발표 대본

다음으로 데이터에 대해 설명드리겠습니다.

저희가 사용한 데이터는 직접 생성한 합성 이미지 데이터입니다. 외부에서 복잡한 대용량 데이터를 가져오는 대신, 프로젝트 목적에 맞게 원형, 삼각형, 사각형 표지판 이미지를 직접 만들었습니다. 이렇게 한 이유는 데이터 구조를 명확히 이해할 수 있고, GitHub에 코드와 데이터를 함께 올리기에도 적합하기 때문입니다.

데이터는 총 3개 클래스로 구성되어 있습니다. 원형 표지판은 circle, 삼각형 표지판은 triangle, 사각형 표지판은 square로 분류했습니다. 각 클래스마다 160장씩 이미지를 생성해서 전체 데이터는 480장입니다. 모든 이미지는 64x64 크기의 PNG 파일로 저장했습니다.

이미지를 생성할 때는 단순히 같은 모양을 반복하지 않았습니다. 실제 이미지처럼 약간씩 차이가 생기도록 위치, 크기, 회전, 밝기, 배경 노이즈를 무작위로 적용했습니다. 예를 들어 같은 삼각형이라도 조금 왼쪽에 있거나, 약간 회전되어 있거나, 배경이 조금 흐릿하게 보이도록 만들었습니다. 이렇게 하면 모델이 단순히 하나의 고정된 모양만 외우는 것이 아니라, 형태의 공통적인 특징을 학습할 수 있습니다.

전처리 과정에서는 먼저 이미지를 회색조로 변환했습니다. 이번 프로젝트는 색상보다 형태를 분류하는 것이 목적이기 때문에, 색상 정보는 줄이고 형태 정보에 집중했습니다. 그다음 픽셀 값을 0에서 1 사이로 정규화했습니다. 이렇게 하면 모델이 입력값을 더 안정적으로 처리할 수 있습니다.

마지막으로 64x64 이미지를 32x32 크기의 특징으로 축소했습니다. 이미지 크기를 줄이면 계산량이 줄어들고, 그래도 원형, 삼각형, 사각형을 구분하는 데 필요한 전체적인 형태는 유지됩니다. 전체 데이터 중 75%인 360장은 학습에 사용했고, 25%인 120장은 테스트에 사용했습니다.

### 김수민 발표 대본

이제 모델 구조와 학습 방법을 설명드리겠습니다.

저희는 복잡한 사전학습 모델 대신, 은닉층 1개를 가진 간단한 신경망을 직접 구현해서 사용했습니다. 이번 프로젝트의 목적은 최고 성능을 내는 것보다는 이미지 데이터가 어떻게 전처리되고, 모델이 어떤 방식으로 분류를 수행하는지 이해하는 것이었기 때문에 비교적 단순한 구조를 선택했습니다.

모델의 입력은 전처리된 32x32 이미지 특징입니다. 32x32 이미지는 총 1024개의 픽셀 값으로 표현할 수 있고, 이 값들이 모델의 입력으로 들어갑니다. 모델은 이 입력을 바탕으로 원형, 삼각형, 사각형 중 어떤 클래스에 가까운지 계산합니다.

모델 구조는 입력층, 은닉층, 출력층으로 구성됩니다. 은닉층은 이미지의 형태적인 패턴을 학습하는 역할을 합니다. 출력층에서는 circle, triangle, square 세 클래스에 대한 점수를 계산하고, Softmax 함수를 사용해 이 점수를 확률로 변환합니다.

예를 들어 어떤 이미지에 대해 circle 확률이 0.90, triangle 확률이 0.05, square 확률이 0.05로 나오면 모델은 이 이미지를 원형으로 예측합니다. 반대로 사각형 이미지에서 square 확률이 가장 높으면 사각형으로 분류합니다.

코드 흐름도 간단히 설명드리겠습니다. `generate_dataset.py`는 원형, 삼각형, 사각형 이미지 데이터를 생성하는 코드입니다. `features.py`는 이미지를 불러와 회색조 변환, 정규화, 특징 추출을 수행합니다. `train.py`는 모델을 학습하고 정확도와 혼동행렬을 계산합니다. 마지막으로 `predict.py`는 학습된 모델을 사용해 새로운 이미지 하나를 예측하는 코드입니다.

학습이 끝난 뒤에는 결과 파일이 `outputs` 폴더에 저장됩니다. 여기에는 학습된 모델 파일, 정확도 정보가 들어 있는 `metrics.json`, 혼동행렬 이미지, 그리고 예측 예시 이미지가 포함되어 있습니다. 따라서 GitHub에 올렸을 때 코드뿐만 아니라 데이터와 실험 결과도 함께 확인할 수 있습니다.

### 오영덕 발표 대본

마지막으로 실험 결과와 해석을 설명드리겠습니다.

저희 모델의 실험 결과, 학습 정확도는 100%, 테스트 정확도는 약 94.2%가 나왔습니다. 여기서 학습 정확도는 모델이 학습에 사용한 이미지들을 얼마나 잘 맞혔는지를 의미하고, 테스트 정확도는 처음 보는 이미지에 대해 얼마나 잘 예측했는지를 의미합니다. 따라서 테스트 정확도가 약 94.2%라는 것은 새로운 이미지에 대해서도 비교적 안정적으로 형태를 구분했다는 뜻입니다.

클래스별 정확도를 보면 원형은 약 93.3%, 삼각형은 약 94.9%, 사각형은 약 94.4%가 나왔습니다. 세 클래스 모두 비슷한 수준의 정확도를 보였고, 특정 클래스만 지나치게 낮게 나오지는 않았습니다.

혼동행렬을 보면 대부분의 이미지가 올바르게 분류되었습니다. 다만 일부 원형과 사각형 이미지에서 오분류가 발생했습니다. 특히 사각형이 회전되어 있거나 이미지가 약간 흐려진 경우에는 모델이 원형과 헷갈리는 경우가 있었습니다. 반면 삼각형은 꼭짓점과 변의 방향성이 비교적 뚜렷해서 안정적으로 분류되는 편이었습니다.

이 결과를 통해 단순한 형태 분류 문제에서도 데이터의 다양성과 전처리가 중요하다는 점을 확인할 수 있었습니다. 만약 모든 이미지가 같은 위치와 같은 크기로만 만들어졌다면 정확도는 높게 나올 수 있지만, 실제로는 조금만 달라져도 성능이 떨어질 수 있습니다. 그래서 저희는 위치, 회전, 밝기, 노이즈를 넣어 다양한 상황을 반영하려고 했습니다.

이번 프로젝트의 한계점도 있습니다. 가장 큰 한계는 실제 도로 사진이 아니라 합성 이미지 데이터를 사용했다는 점입니다. 실제 도로에서는 표지판이 가려지거나, 배경이 복잡하거나, 조명이 어둡거나, 카메라 각도가 다를 수 있습니다. 이런 요소는 이번 데이터에 완전히 반영되지는 않았습니다.

개선 방향으로는 실제 교통 표지판 데이터셋을 추가해서 모델을 학습시키는 방법이 있습니다. 또한 현재는 간단한 신경망을 사용했지만, CNN, MobileNet, ResNet 같은 이미지 분류에 더 적합한 모델과 성능을 비교해볼 수도 있습니다. 나아가 표지판의 형태뿐만 아니라 색상, 글자, 내부 아이콘까지 함께 인식한다면 더 실용적인 교통 표지판 인식 모델로 확장할 수 있습니다.

결론적으로 저희는 이번 프로젝트를 통해 이미지 데이터 생성, 전처리, 모델 학습, 평가, 결과 해석까지 컴퓨터 비전 프로젝트의 전체 과정을 경험했습니다. 단순한 형태 분류에서 시작했지만, 이를 기반으로 실제 표지판 인식 문제로 확장할 수 있다는 점에서 의미가 있다고 생각합니다.
